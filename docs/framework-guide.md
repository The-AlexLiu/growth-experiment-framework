# 增长实验框架使用指南

这份指南用于把一个模糊的增长目标拆成可执行实验。核心原则是：先定义问题，再写假设，最后设计动作和判断标准。

## 一次实验必须回答的 6 个问题

| 问题 | 说明 |
|---|---|
| 业务问题是什么 | 当前要解决的是获客、转化、复购、留存还是规模化问题 |
| 目标用户是谁 | 不要只写泛人群，要写清场景和需求强度 |
| 增长假设是什么 | 用“如果...那么...”表达可验证假设 |
| 测试动作是什么 | 明确要测的渠道、素材、落地页、优惠或转化路径 |
| 核心指标是什么 | 一个实验优先选择 1-2 个主指标 |
| 成功后下一步是什么 | 提前定义放大、复测、二创、暂停或补数据动作 |

## 推荐假设句式

```text
如果 [目标用户] 在 [触发场景] 下对 [核心痛点] 高度敏感，
那么 [测试动作] 应该让 [核心指标] 达到 [成功标准]。
```

示例：

```text
如果新手妈妈在夜间喂养场景下对省时和安全感高度敏感，
那么围绕夜间痛点的短视频素材应该让 CTR 高于账户近 14 天均值 20%。
```

## 指标选择建议

| 实验目的 | 优先指标 | 辅助指标 |
|---|---|---|
| 验证内容吸引力 | CTR / Hook Rate | CPM / CPC / 互动率 |
| 验证用户兴趣 | LPV / ATC / IC | 页面停留、跳出率 |
| 验证转化效率 | CVR / CPA / ROAS | AOV / 支付转化率 |
| 验证可放大性 | 花费规模 / ROAS 稳定性 | 素材生命周期 / 频次 |

## 常见错误

| 错误写法 | 问题 | 更好的写法 |
|---|---|---|
| 测试素材 | 没有假设，无法复盘原因 | 测试 3 条痛点素材，验证痛点内容是否提升 CTR |
| 看 ROAS 好不好 | 指标太后置，早期样本不足 | 先看 CTR、ATC、CPA，再判断是否继续放量 |
| 用户喜欢便宜 | 太泛，无法指导内容 | 价格敏感用户是否更容易被限时权益触发 |
| 跑不动就换受众 | 未判断是素材问题还是受众问题 | 先拆 CTR、CVR、CPA，再决定优化方向 |

## 最小可执行流程

1. 写清业务阶段和核心目标。
2. 列出 3 个用户 / 场景假设。
3. 为每个假设设计 1 个可测试动作。
4. 每个实验只指定 1-2 个核心指标。
5. 提前写清成功标准和失败后的下一步。
6. 实验结束后记录结论，而不是只记录结果。
