# 增长实验复盘检查清单

用于实验结束后判断下一步动作，避免只看结果、不看原因。

## 基础信息

| 检查项 | 判断标准 |
|---|---|
| 实验编号是否唯一 | 便于追踪和复盘 |
| 实验周期是否明确 | 至少记录开始日期和结束日期 |
| 测试动作是否清楚 | 能看出具体测了什么 |
| 核心指标是否提前定义 | 不在实验结束后临时换指标 |
| 成功标准是否提前定义 | 避免主观解释结果 |

## 数据判断

| 结果表现 | 优先检查 |
|---|---|
| CTR 高，CVR 低 | 落地页、信任信息、价格、权益、页面速度 |
| CTR 低，CVR 高 | Hook、素材角度、封面、前 3 秒表达 |
| ATC 高，购买低 | 运费、支付方式、折扣门槛、售后承诺 |
| CPA 低但量小 | 受众规模、预算、素材生命周期、竞价限制 |
| ROAS 高但不稳定 | 样本量、归因窗口、单日异常订单、复购影响 |

## 下一步动作分类

| 分类 | 适用情况 |
|---|---|
| 立即放大 | 核心指标达标，样本量足够，转化链路无明显异常 |
| 小预算复测 | 数据方向不错，但样本量不足或波动较大 |
| 优化后再测 | 找到明确短板，例如 Hook 弱、证言弱、落地页弱 |
| 暂停实验 | 核心指标明显不达标，且没有清晰优化方向 |
| 补充数据 | 当前数据口径不一致、归因不清或关键字段缺失 |

## 复盘输出格式

建议每个实验结束后输出：

```text
实验编号：
实验假设：
测试动作：
核心数据：
是否达标：
主要结论：
下一步动作：
需要补充的信息：
```
