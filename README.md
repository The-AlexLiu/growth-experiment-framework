# Growth Experiment Framework

**增长实验框架：把“要增长”拆成可以验证的假设和动作**

做新品、新业务或新渠道增长时，最常见的问题不是没有动作，而是动作太散。

今天想测素材，明天想换受众，后天又觉得落地页不行。每个动作看起来都合理，但如果背后没有清楚的假设，最后复盘时就很容易只剩一句话：

> 这次效果不好，下次再试试别的。

这其实很可惜。因为增长实验最有价值的地方，不只是某一次结果好不好，而是每一次测试之后，团队是否更清楚：

- 谁更可能需要这个产品？
- 用户在什么场景下会产生需求？
- 哪个痛点更容易触发行动？
- 哪种内容更能建立信任？
- 哪个环节阻碍了转化？
- 下一轮应该放大、复测、暂停，还是补数据？

所以我做这套框架时，想解决的不是“写一份漂亮的实验计划”，而是把模糊的增长目标拆成一组可以被验证、被复盘、被继续推进的动作。

## 我为什么要做这套框架

很多增长问题一开始都很模糊。

比如：

- 新品上线后不知道先投哪个渠道；
- 素材跑了几条，但不知道为什么有的好、有的差；
- 落地页改了几版，但不知道到底影响了哪个环节；
- 广告花费有了，但转化不稳定；
- 团队复盘时只讨论结果，没有沉淀原因。

这些问题如果直接进入执行，很容易变成“多做一点、多投一点、多试一点”。但增长不是靠动作堆出来的，尤其在预算有限、样本有限、历史数据不足的时候，更需要先把问题拆清楚。

我更倾向于先问：

> 这次增长动作到底想验证什么？

只要这个问题没有回答清楚，后面的素材、渠道、落地页、优惠、预算和复盘都容易失焦。

---

## 我的核心判断

一个增长实验不应该只写“测试 Meta 广告”或“测试一批素材”。这类描述只能说明动作，不能说明判断逻辑。

我更希望每个实验都能说清楚这 6 件事：

| 问题 | 我真正想确认什么 |
|---|---|
| 业务问题是什么 | 当前要解决的是获客、转化、复购、留存还是规模化 |
| 目标用户是谁 | 不是泛人群，而是具体场景下有需求的人 |
| 用户为什么会行动 | 痛点、动机、触发场景和购买理由是什么 |
| 我要测试什么 | 渠道、素材、内容角度、落地页、权益或转化路径 |
| 用什么指标判断 | CTR、ATC、CVR、CPA、ROAS、GMV 等指标各自回答什么问题 |
| 结果出来后怎么办 | 成功后放大，失败后优化、复测、暂停还是补数据 |

如果这 6 件事没有写清楚，实验就很难复盘。因为你只能知道“结果如何”，却很难知道“为什么是这个结果”。

## 我会把实验当成一套运营系统

如果只把增长实验理解成“每周想几个测试点”，团队很快会遇到三个问题：

1. 实验越来越多，但不知道哪些真正服务业务目标；
2. 每个实验都说自己重要，但资源永远不够；
3. 复盘只停留在单次结果，没有形成可复用判断。

所以我更倾向于把增长实验管理成一个小型运营系统：

```text
业务目标 -> 指标拆解 -> 机会池 -> 优先级排序 -> 实验设计 -> 执行监控 -> 复盘沉淀 -> 下一轮策略
```

这个系统的重点不是让团队“多做实验”，而是让每一次实验都能回答一个关键业务问题。

### 1. 先做增长目标树

我会先把增长目标拆到具体业务杠杆：增长来自获客、转化、客单、留存还是复购，哪个环节最值得先验证。

| 层级 | 示例问题 | 对应指标 |
|---|---|---|
| 北极星指标 | 业务最终要提升什么 | GMV、有效线索、付费用户、复购订单 |
| 一级驱动 | 增长来自获客、转化、客单、留存还是复购 | 新客数、CVR、AOV、复购率 |
| 二级杠杆 | 哪个环节最可能被运营动作影响 | CTR、LPV、ATC、IC、支付转化率 |
| 实验动作 | 具体要测什么 | 渠道、素材、落地页、权益、触达节奏 |

如果目标树没拆清楚，实验很容易变成“哪里都试一点”。这类实验即使偶尔跑出结果，也很难复制。

### 2. 建机会池，减少临时起意

实验入口我会放在机会池里：先收集真实信号，再筛选值得验证的机会。

我会把机会分成几类：

| 机会类型 | 典型来源 | 示例 |
|---|---|---|
| 用户洞察 | 评论、客服、问卷、访谈、社媒内容 | 用户担心效果不真实 |
| 数据异常 | 漏斗、渠道、素材、页面数据 | CTR 高但 CVR 低 |
| 渠道机会 | Meta、Google、TikTok、SEO、KOL | 搜索词显示某个场景需求强 |
| 内容机会 | 爆款素材、竞品内容、达人反馈 | 对比演示素材互动率高 |
| 产品/Offer | 价格、套装、赠品、物流、售后 | 免邮门槛可能影响支付 |
| 生命周期 | 新客、老客、沉默用户、复购用户 | 购买 30 天后可推补充装 |

机会池的作用，是让实验来自真实信号，而不是来自拍脑袋。

### 3. 用优先级决定先做什么

实验进入排期前，我会先判断这件事是否值得投入资源。

我会用一个简化评分：

| 维度 | 问题 | 评分 |
|---|---|---:|
| Impact | 如果成功，对核心指标影响大不大 | 1-5 |
| Confidence | 有没有数据、用户反馈或历史经验支撑 | 1-5 |
| Effort | 需要多少人力、素材、开发、预算 | 1-5，越低越好 |
| Speed | 多快能上线并拿到初步结果 | 1-5 |
| Risk | 是否可能伤害品牌、利润或用户体验 | 1-5，越低越好 |

优先级不是为了追求数学精确，而是为了让团队在资源有限时有共同判断标准。

我通常会优先做：

- 影响核心指标；
- 有真实用户或数据证据；
- 上线成本低；
- 可以快速得到反馈；
- 即使失败也能沉淀结论的实验。

### 4. 明确实验准入标准

很多实验失败，不是因为方向错，而是因为还没准备好就上线。

一个实验进入执行前，至少要过这几个门槛：

| 准入项 | 判断标准 |
|---|---|
| 业务问题明确 | 能说明当前卡点是获客、转化、复购还是放量 |
| 假设可验证 | 能用“如果...那么...”写清楚 |
| 动作可执行 | 渠道、素材、页面、预算、负责人明确 |
| 指标可追踪 | 核心指标和辅助指标能拿到 |
| 成功标准提前写好 | 不能等结果出来后再改口径 |
| 风险可控 | 不会明显伤害品牌、毛利、库存或用户体验 |

如果这些写不清楚，我会先把它放回机会池，而不是硬推上线。

### 5. 建立实验节奏

增长实验需要节奏，不是想到就做、做完就忘。

一个比较稳的节奏是：

| 节奏 | 重点 |
|---|---|
| 周初 | 确认本周实验排期、预算、素材、页面和负责人 |
| 周中 | 只看异常和执行质量，不急着下最终结论 |
| 周末 | 复盘数据，给出放大、复测、优化、暂停或补数据结论 |
| 月度 | 汇总结论，更新机会池、素材库、页面优化方向和渠道策略 |

这个节奏能避免两个极端：一是每天情绪化改实验，二是实验跑完没人复盘。

## 我会怎么拆一个增长问题

### 第一步：先定义业务问题

拆增长问题时，我通常先判断当前业务卡点，再决定渠道、素材、页面或活动动作。

| 业务阶段 | 典型问题 | 更适合先验证什么 |
|---|---|---|
| 新品冷启动 | 不知道有没有需求 | 用户痛点、场景、内容角度 |
| 早期验证 | 有点击但转化不稳 | 信任、价格、落地页、购买阻力 |
| 渠道测试 | 不知道哪个渠道更有效 | 渠道意图、流量质量、转化成本 |
| 素材测试 | 不知道什么内容能带来行动 | Hook、证言、卖点、CTA |
| 放量阶段 | 小预算有效，大预算跑不动 | 素材规模、受众扩展、频次、边际成本 |

这一步的重点是先把问题范围缩小。否则“增长不好”这个问题太大，任何动作都像是可能的解决方案。

### 第二步：把用户和场景说清楚

新品增长的第一步，是找到“谁在什么情况下真的需要它”。

我会把用户假设拆成这几个字段：

| 用户人群 | 核心痛点 | 触发场景 | 购买动机 | 购买阻力 | 内容角度 |
|---|---|---|---|---|---|
| 示例：高频使用某类产品的人 | 示例：当前方案效率低或体验差 | 示例：遇到具体任务或问题时 | 示例：希望更省时、更稳定、更有安全感 | 示例：担心价格、效果、售后或学习成本 | 示例：痛点前置、结果展示、真实证言 |

如果这张表写不出来，说明现在还不适合直接放大投放。因为你还不知道该对谁说、说什么、为什么对方会信。

### 第三步：把动作写成可验证假设

我会尽量避免只写：

```text
测试素材
```

更好的写法是：

```text
如果目标用户在某个场景下对某个痛点高度敏感，
那么围绕这个痛点的内容应该在点击率、互动率或加购率上高于普通卖点内容。
```

这个写法的好处是，实验结束后你能判断的是“痛点内容是否成立”，而不只是“这几条素材好不好”。

### 第四步：给指标分层

我会把最终 ROAS 放在完整链路里看。尤其在新品早期，样本少、预算小、学习期波动大，只看最终转化很容易误判。

| 层级 | 常见指标 | 主要回答什么 |
|---|---|---|
| 前链路 | CTR / Hook Rate / 互动率 | 内容是否能吸引用户 |
| 中链路 | LPV / ATC / IC | 用户是否产生兴趣 |
| 后链路 | CVR / CPA / ROAS | 是否具备转化效率 |
| 业务层 | GMV / 毛利 / 回本周期 | 是否值得继续放大 |
| 内容层 | 素材生命周期 / 复用率 | 是否能沉淀内容资产 |

指标不是越多越好。一个实验最好有 1 到 2 个核心指标，再配少量辅助指标，否则复盘会变成到处找理由。

### 第五步：提前写清下一步动作

很多实验的问题不是没有结果，而是结果出来后不知道怎么办。

所以我会在实验开始前就写清楚：

- 达到什么标准就放大；
- 没达到但有局部信号时怎么复测；
- 哪些情况说明要暂停；
- 哪些情况说明不是素材问题，而是落地页、价格、受众或信任问题；
- 数据不足时还需要补什么。

这样复盘时可以跳出“好 / 不好”的简单判断，直接进入下一轮动作。

## 实验决策规则

实验复盘时，我会避免只写“成功 / 失败”。更好的做法是把结果分成 5 类：

| 结论 | 判断标准 | 下一步 |
|---|---|---|
| 放大 | 核心指标达标，样本量足够，执行无明显异常 | 加预算、扩渠道、扩素材或扩大人群 |
| 复测 | 方向有信号，但样本不足或波动较大 | 控制变量再跑一轮 |
| 优化后再测 | 找到明确短板 | 改素材、页面、权益或路径后再测 |
| 暂停 | 核心指标明显不达标，且没有局部信号 | 停止投入，记录原因 |
| 补数据 | 数据口径、埋点、样本或归因不清 | 先修数据，再判断 |

这套规则的价值，是减少情绪化判断：不因为一次波动就放大，也不因为一次失败就丢掉有价值的方向。

## 常见复盘判断

实验结束后，我会尽量避免只说“表现好”或“表现差”。更有价值的是把结果和可能原因对应起来。

| 结果表现 | 可能原因 | 下一步动作 |
|---|---|---|
| CTR 高，CVR 低 | 内容吸引人，但信任、价格或落地页不足 | 优化落地页、补充证言、调整权益 |
| CTR 低，CVR 高 | 内容吸引力弱，但进入后的用户意图强 | 优化 Hook，扩展内容角度 |
| ATC 高，购买低 | 用户有兴趣，但购买阻力强 | 优化价格、物流、支付、信任信息 |
| ROAS 高但跑不动 | 素材或受众规模有限 | 扩展素材矩阵，测试更多场景 |
| CPC 高但 CVR 好 | 流量贵但意图强 | 评估毛利和规模，不急着否定 |
| 花费少、数据波动大 | 样本不足 | 延长观察或降低结论强度 |

复盘不是为了证明某个人判断对错，而是为了让下一轮测试更清楚。

## 适用场景

- 新品冷启动；
- 新业务增长验证；
- 内容增长实验；
- 用户获取策略设计；
- 广告素材测试；
- 营销活动复盘；
- 渠道策略拆解；
- 落地页承接与转化路径优化；
- 增长团队内部协作。
